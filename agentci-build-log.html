Plan 06 — Compounding Regression Immunity DONE

sprint · D15–D19 · 3 implementer subagents + controller, opus review

What it is

A closed self-improving loop on top of the existing regression CI. When the gate goes red, an agent authors a "guard" — a targeted regression test — that, once human-approved, becomes a permanent member of the eval suite so the same regression can never silently pass twice.

Decisions locked (D15–D19)

  • D15 — Agent-authored hybrid guards: deterministic assertion and/or scoped LLM-judge rubric, admitted only via a two-sided discrimination test (must FAIL on the regressed answer, PASS on the gold/known-good answer).
  • D16 — Guard gate (behavior C): tripping a persisted guard is an instant RED (verdict="guard_blocked") with the investigator naming which learned guard tripped.
  • D17 — Frozen cross-family improvement ruler: held-out lift AND the pass→fail regression leg are scored by a non-Gemini judge (claude-haiku-4-5), so the Gemini optimizer cannot grade its own homework.
  • D18 — Independent rubric review: adversarial review of candidate guards by the cross-family model.
  • D19 — Diagnose/fix split: diagnosis (+ guard authoring) and fix-authoring are separate agents — a lesson borrowed from LangChain's LangSmith Engine.

Design rationale

This plan answers an architectural question revealed in post-Plan-05 reflection: if we're treating Agent CI as a closed-loop optimizer (D11/D12), how do we prevent the optimizer from gaming its own evaluators, and how do we build permanent institutional memory against repeated regressions? The discrimination-test framing and two-sided guard validation are stronger than LangSmith Engine's one-sided evaluator check; the cross-family judge (D17) prevents self-grading; and the diagnose/fix split (D19) enforces specialization so root-cause reasoning doesn't contaminate fix engineering. The designs were validated against Engine's published architecture.

What was built

  • agentci/engineer/guard.py — guard runner, discrimination test, and persisted-guard loader. Guards are stored as Guard dataclass (id, author-date, guard-case-id, trigger-phrase, assertion/rubric pair).
  • agentci/engineer/independent_judge.py — cross-family judge using claude-haiku-4-5 for lift scoring and regression-leg scoring (D17).
  • agentci/engineer/review.py — adversarial rubric reviewer (D18). Evaluates: does the guard trigger on the regressed answer? Does it pass on the gold? Is it overfit (does it reject unrelated answers)?
  • agentci/engineer/diagnose.py + fix_author.py — split agent tasks replacing old investigate.py (kept but unused by run_check). diagnose.py drives the agent through root-cause reasoning + guard authoring; fix_author.py proposes the fix separate from diagnosis (D19).
  • Modified: config.py (added haiku model + guard config), lift.py (replaced in-family judge with independent judge, D17), mint.py (persist guards on human approve), report.py (surface guard info, "guard_blocked" verdict), engineer/__init__.py (new run_check orchestration with guard gate check + diagnose/fix separation).
