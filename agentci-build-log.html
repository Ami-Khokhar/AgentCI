Bug found & fixed in the plan itself DEVIATION

The implementer correctly refused to silently alter spec'd code and flagged that Task 5's spec'd test and spec'd implementation contradict each other:

  • Test asserts an unknown query ("how to fly to the moon") returns sections == [].
  • But the spec'd matcher used naive substring matching over every query token, so stop-words like "to" / "the" matched every KB body → never empty. Test could never pass.

Reasoning for the fix

This is not one of the frozen decisions (D1–D10) — it's a bug in the plan's example code, and the plan's own "done" bar is all-green pytest. The intent is unambiguous (unknown query → no sections). I applied the minimal intent-preserving fix: drop stop-words and ≤2-char tokens before matching. The function's return contract {"status","sections"} is unchanged. SendMessage isn't available in this harness and the fix is a fully-understood 4-line change, so I applied it directly rather than spending a fresh subagent on it. Committed separately (0ac8417) so the deviation is auditable.
